Overview

ShareTab is a collaborative browser client-server application. It was originally developed for linux, so ShareTabWin is the name of the Windows version of ShareTab.

ShareTab allows mutiple users to get together over a network and work together in order to research information on the internet. They can surf individually (in private) or in a common session led by one of them.

ShareTabWin targets the .NET 4.0 platform from Microsoft, on systems where this is not available it must be installed before running ShareTabWin.

Development environment

ShareTab was written using Microsoft Visual Studio 2010 Professional. Parts of it were visually designed using Microsoft Expression Blend and Microsoft Expression Design.

The language used everywhere in the project is C#. The client part targets the WPF, while the client–server communication part targets the WCF.

The UI programming is event driven to ensure extensibility, and binding driven for clean code. For example, everytime the client receives an update from the server, a particular event is raised. The controller for the particular client-side model handles the event by updating the collection (adding the newly connected user, for example). The user interface is bound to the collection and automatically updates when it changes.

Structure

ShareTabWin is structured in 3 projects:

Infrastructure
The common datatypes used for communicating between the client and the server. This is implementation agnostic, but all data needs to be marked as serializable.
Communication
Contains the WCF architecture of the ShareTab server and the ShareTab client components. By extending the base WCF service classes this offers a robust communication protocol.
ShareTabWin
The ShareTab graphical user interface and interaction code. Apart from the other projects mentioned above, this depends on outside libraries such as AvalonDock, GeckoFX, xulrunner (more in the next section)

Components

ShareTab depends on the following components:

AvalonDock
Docking manager for WPF with autohide and floating capabilities. Hosted at http://avalondock.codeplex.com/.
GeckoFX
WinForms browser control based on Gecko and xulrunner. A very unstable project, causes most of our bugs. Available at http://geckofx.org/
